Config files reference

Section 22 — what each file controls and what loads it. Edits: after changing YAML, restart the app — ConfigManager caches each file after the first read in a process.

Config file Controls Who reads it
config/regimes.yaml ADX thresholds, vol percentiles, regime tradability systems/regime/service.py, systems/regime/backend.py, regime UI
config/strategy_registry.yaml 208 strategies, enabled, status, priors systems/strategy_router/service.py
config/risk_rules.yaml Base risk %, max DD, spread percentile threshold systems/risk/backend.py, systems/risk/shield.py, research
config/funded_rules.yaml Daily loss limit, trailing DD, prop-style caps systems/risk/backend.py → systems/risk/funded_rules_engine.py
config/sessions.yaml Kill zone GMT windows, session labels Loaded into classify_session(..., sessions_config) in core/time_utils.py; YAML loaded by systems/regime/service.py, systems/data/service.py, systems/research/service.py
config/symbols.yaml pip size, pip_value_per_lot, swap, lot limits systems/risk/position_sizer.py
config/app.yaml App name, paper_trading_enabled, live_trading_enabled, defaults app/main.py, systems/monitoring/backend.py, Settings UI
systems/regime/config.yaml Feature lookback periods (regime subsystem) systems/regime/service.py
systems/risk/config.yaml Duplicate risk keys (not wired) Runtime uses config/risk_rules.yaml — keep this file in sync for documentation or remove to avoid drift.
config/strategy_router.yaml Registry validation (strict_blueprint, etc.) systems/strategy_router/service.py
config/signal_routing.yaml Per-regime signal codes, size multipliers / overrides systems/strategy/signal_routing.py
Strategy & live flags
  • Change enabled and status on registry rows only after backtests satisfy the enablement gate.
  • live_allowed on a strategy: manual operator only. Never set via an AI prompt.

Edit configs from /settings where exposed, or on disk in config/.
