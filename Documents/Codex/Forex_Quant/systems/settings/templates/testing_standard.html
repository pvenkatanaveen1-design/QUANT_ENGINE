QUANTA V3 master prompt — alignment (implemented vs deferred)

This app follows QUANTA_V3_COMPLETE_MASTER_PROMPT.md build order for Phases A–G where coded. Deferred by design: Phase H (paper broker / paper_execute wiring), Phase I (order_send), full HMM (Phase G note). forex_regime/regimes52 remains a parallel tree — do not merge into the control-center path.

  • A–B: Wilder ADX, SMA-seeded EMA, Q2 before forced Q4 on trend+high-vol, session gaps covered in config/sessions.yaml, eight signal functions + compute_signal, decision engine uses signals (research mode → research_only).
  • C–D: Data layer + walk-forward / signal-engine backtest → DuckDB cache + equity snapshots.
  • E–F: Journal JSONL + SQLite decisions; analysis schema + 12 cache types (incl. WF_RUN_SUMMARY).
  • G: /analytics — ten visualization groups from cache + KPIs (Section 12).
  • Section 17: eight core pages + aliases (/monitor, /research); extra nav (MT5, settings, testing standard, etc.) does not replace them.
  • Observability: slope_score (OLS slope / ATR) is on RegimeFeatureSet for UI/diagnostics; quadrant logic still uses ER + ADX + vol thresholds only.
Mappings from the master prompt
  • result.adx_from_features() → result.features.adx
  • calculate_lot_size(..., cfg) → calculate_lot_size(..., "EURUSD", ConfigManager) optional; tests use default project root.
  • risk_pct = 0.01 for ~0.05 lots → use 0.001 (0.1% risk); 0.01 = 1% → ~0.5 lots at same stop/pip value.
  • Choppy OHLC alone often spikes volatility_percentile → Q4; the Q3 test calls _choose_base with a hand-crafted RegimeFeatureSet to validate the range bucket.
